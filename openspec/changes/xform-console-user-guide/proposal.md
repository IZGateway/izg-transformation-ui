# Proposal: Xform Console User Guide

**Jira:** [IGDD-2517](https://izgateway.atlassian.net/browse/IGDD-2517)
**CR Name:** `xform-console-user-guide`
**Sprint:** Sprint 33 — Xform Advanced Features

## User Story

As a Solutions Engineer or IZ Gateway administrator, I want a task-based user guide
for the Xform Console — with screenshots showing exactly what I will see — so that I
can learn and reference the application without depending on team support.

## Problem Statement

The Xform Console has no end-user documentation. [IGDD-1813](https://izgateway.atlassian.net/browse/IGDD-1813)
proposed adding a documentation page in the UI; the team resolved it Won't Do, with the
decision that docs should live in GitHub alongside the code. No follow-through occurred
until this ticket.

Users currently have no self-service reference for any Xform Console workflow. Every
question requires direct team contact.

## What the Guide Must Provide

The guide is derived from existing Jira tickets and the codebase — not written from
scratch. For each feature area:

1. **User story** — drawn from the originating Jira ticket (e.g., IGDD-2533, IGDD-2697,
   IGDD-2698, IGDD-2699)
2. **Step-by-step task instructions** — written from the user's perspective, based on
   the implemented UI
3. **Success criteria** — what the user sees when the task completes correctly, derived
   from ticket acceptance criteria
4. **Failure cases** — what error messages or states the user encounters when something
   goes wrong, and what to do
5. **Screenshots** — actual UI captures for each major step and state, generated by a
   Playwright capture script committed alongside the guide

The screenshots must be reproducible: anyone with access to a running Xform Console
instance can re-run the capture script to refresh images after a UI change.

"Coming in a future release" placeholder so the structure is ready to fill in as
each feature ships.

## Non-Functional Requirements

- Format: GitHub Flavored Markdown; lives in `public/help/` in `izg-transformation-ui`
- Entry point: `public/help/index.md` (table of contents + links to feature sections)
- Screenshots: PNG images in `public/help/images/`, generated by a committed Playwright
  capture script (Chromium only; no assertions; not a regression test suite)
- Linked from the repository `README.md`
- Structured to support incremental expansion as new features are released
- **Lite in-app help**: A `HelpPanel` component (MUI `<Drawer>` + `react-markdown`)
  renders the relevant guide section when a Help button is clicked on each page.
  The same Markdown files in `doc/` serve both the standalone guide and the in-app
  panel — no duplication. This is a lightweight Help button per page, not a full
  in-app documentation site (distinguishing this from the Won't Do decision in
  IGDD-1813, which was about a dedicated documentation route in the UI).

## Out of Scope

- Full in-app documentation site or dedicated documentation route (closed Won't Do, IGDD-1813)
- Confluence-hosted documentation
- API reference documentation
- Developer/contributor documentation (README.md covers this)
- Full Playwright regression test suite (tracked separately as [IGDD-2887](https://izgateway.atlassian.net/browse/IGDD-2887))
